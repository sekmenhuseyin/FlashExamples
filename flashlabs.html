Flash 5 & Flash MX, Flash 8 examples

These are projects to implement familiar games and other activities, such as surveys. Some are similar to those in the Visual Basic text & course. In addition, there are projects giving instructions for folding origami models. As indicated, some have tutorial material (instructions on how to build the game) suitable for self-study or a lab in a course. A discussion comparing Visual Basic and Flash, focusing on cannonball, is given in Comparison of Visual Basic and Flash for implementing a cannonball (ballistics) simulation .  For JavaScript examples, including games, try JavaScript Labs .

Preparation: the assumption here is that you have some knowledge of Flash basics such as can be acquired by doing the built-in lessons.  The ActionScript is done using the (so-called) Expert Mode, not the Normal Mode and not with Script Assist.  To change to Expert Mode, click on the icon in the upper right corner of the Actions panel.  
  • The practice of attaching code to instances on the Stage, such as on(Press),onClipEvent(enterframe), key events, etc., is now declared a bad practice. Instead, code is attached (that is, event handlers are specified) using code. While recognizing the problem in finding code, the old way had appeal. One advantage of the new way, however, is in avoiding creating objects just to 'listen'.
    The following examples show the difference. These are two implementations of a silly simulation of birds pecking for worms. The birds jump around and if they collide with worms, the worm is eaten. You click on the key b to create a new bird and the key w to create a new worm. You click directly on a bird to change the size of the jump and directly on a worm to change the size of the worm. Birds that eat worms grow in size. Click the Help button to start (this gets the focus).

  • Work in progress: Sudoku solver: play yourself and/or get help: heuristics, backtracking, save and restore running program and to disk (using SharedObject). Application does not supply starting patterns. You enter a pattern and play game and/or use program features. The try... uses heuristics and, if there is no progress, guesses, with the possibility of backtracking. This is not fully tested. The small numbers represent the computed possibilities for each cell. Saving is done using SharedObject, a Flash facility similar to cookies.
